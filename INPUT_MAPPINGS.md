# Input Mapping Guide

This document outlines the input mappings for the Lumina Reader, ensuring a consistent experience across touch, mouse, and keyboard interactions.

## Navigation Zones (Touch & Mouse)

The screen is divided into three vertical zones:

| Zone | Width | Action |
|------|-------|--------|
| **Left** | 0% - 30% | Go to Previous Page |
| **Center** | 30% - 70% | Toggle Header & Controls |
| **Right** | 70% - 100% | Go to Next Page |

*Note: Clicking/Tapping on links or buttons takes precedence over these zones.*

## Keyboard Shortcuts

| Key | Action |
|-----|--------|
| **Arrow Right** | Next Page |
| **Space** | Next Page |
| **Arrow Left** | Previous Page |
| **Shift + Space** | Previous Page |
| **Enter** | Toggle UI |
| **Escape** | Toggle UI |

## Touch Gestures

| Gesture | Action |
|---------|--------|
| **Tap Left Zone** | Previous Page |
| **Tap Right Zone** | Next Page |
| **Tap Center Zone** | Toggle UI |
| **Swipe Right** (>50px) | Previous Page |
| **Swipe Left** (>50px) | Next Page |

## Visual Feedback

All interactions (Touch, Mouse, Keyboard) trigger a visual ripple effect at the point of interaction (or screen center for keyboard) to confirm the action.
